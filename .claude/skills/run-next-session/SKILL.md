# Run Next Session

## 1. Найди следующую сессию
Прочитай CLAUDE.md, найди первую сессию в разделе "## 6. Сессии" без отметки ✅ в заголовке.

## 2. Выполни ТЗ
Полностью выполни блок "ТЗ для Claude Code" из найденной сессии:
- Создай/измени код
- Применяй миграции через `npx supabase db push`
- Коммит и push (используй `git push origin claude/...:main` для worktree-веток)

## 3. Автопроверки кода
- `cd backend && npm run build`
- `cd frontend && npm run build`
- `git log --oneline -1` — последний коммит должен быть от этой сессии
- Если в сессии была миграция: `npx supabase db push --dry-run` должен вернуть "up to date"

При падении автопроверки — две попытки починить. Если не получилось — запиши проблему в TEST_REPORT.md (раздел "Найденные баги, не починил"), НЕ ставь ✅ и переходи к шагу 5.

## 4. E2E проверка через Playwright
Перед началом проверки убедись что Dev Mode включён: POST /api/team/admin/dev-mode с {enabled: true, hours: 12}. Если уже включён — пропусти.

Определи какая страница/функциональность менялась в этой сессии и проверь её через Playwright:
- Открой соответствующий URL на https://potok-omega.vercel.app
- Выполни 2-3 базовых сценария (создать, открыть, сохранить, что применимо)
- Проверь: нет ошибок в console, API возвращает 200, страница отрисовалась без поломок

Если что-то нельзя проверить автоматически (требует Telegram-бота, реальной задачи с агентом, специфического файла, ручного восприятия "красиво/некрасиво") — запиши в TEST_REPORT.md в раздел "Требует ручной проверки":
"## Сессия N — нужна ручная проверка
URL: [где проверять]
Что сделать: [пошагово, что открыть и нажать]
Что должно произойти: [ожидаемый результат]"

Если E2E нашёл баг — попробуй починить, перепроверь. Если не вышло за 2 попытки — запиши в TEST_REPORT.md (раздел "Найденные баги, не починил").

## 5. Отметка о завершении
Если автопроверки кода прошли (E2E может быть частичный):
- В заголовке сессии в CLAUDE.md добавь ✅ и сегодняшнюю дату: `### Сессия N — Название ✅ 2026-MM-DD`
- Если были отклонения от ТЗ — добавь строку `**Отклонения:** ...` после "Критерии готовности"
- Закоммить CLAUDE.md и TEST_REPORT.md одним коммитом

Если автопроверки кода упали и не починил — НЕ ставь ✅, но коммить TEST_REPORT.md.

## 6. Финал
Напиши краткое сообщение: "Сессия N — ✅ готова" или "Сессия N — не прошла проверки, см. TEST_REPORT.md".
