# Деплой Потока — пошаговая инструкция

Деплоим в два места:
- **Фронтенд** (`frontend/`) → **Vercel**
- **Бэкенд** (`backend/`) → **Railway**

Перед началом: код уже залит на GitHub в репозиторий `tokarevexplorer-svg/potok`. Аккаунты Vercel и Railway уже созданы. Все ключи (Supabase, Apify, OpenAI) уже под рукой — те же, что в локальных `.env`.

Идём по порядку: сначала бэкенд (ему не нужен URL фронтенда), потом фронтенд (ему нужен URL бэкенда).

---

## Часть 1. Бэкенд на Railway

### Шаг 1.1 — создать проект из GitHub

1. Открой [railway.com/new](https://railway.com/new) (войди через GitHub, если ещё не входил).
2. Жми **«Deploy from GitHub repo»**.
3. Если Railway не видит репозиторий — нажми **«Configure GitHub App»** и дай доступ к репозиторию `potok`. Вернись и выбери его.
4. Railway создаст проект и сервис. **Не запускай деплой сразу** — сначала настроим Root Directory и переменные.

### Шаг 1.2 — указать, что код бэкенда лежит в подпапке

1. В созданном сервисе → вкладка **Settings**.
2. Найди раздел **Source** → поле **Root Directory**.
3. Введи: `backend`
4. Нажми **«Update»** (или галочку рядом).

Это говорит Railway: «бери только папку `backend/`, не весь репозиторий». Без этого билд провалится — Railway не поймёт, какой `package.json` использовать.

### Шаг 1.3 — переменные окружения

В том же сервисе → вкладка **Variables** → **«+ New Variable»**. Добавь по одной (значения возьми из локального `backend/.env`):

| Имя | Значение |
|---|---|
| `APIFY_API_TOKEN` | твой токен Apify |
| `APIFY_ACTOR_ID` | `apify/instagram-scraper` (если хочешь шеры — `apify/instagram-reel-scraper`) |
| `SUPABASE_URL` | твой URL из Supabase Project Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | **service role**, не anon. Из того же раздела API |
| `OPENAI_API_KEY` | твой ключ OpenAI |
| `OPENAI_WHISPER_MODEL` | `whisper-1` (опционально, если хочешь по дефолту — пропусти) |
| `OPENAI_ANALYSIS_MODEL` | `gpt-4o-mini` (опционально) |
| `WORKER_CONCURRENCY` | `2` (можно поднять до 3–4 позже) |
| `FRONTEND_ORIGINS` | пока пусто — заполним после деплоя Vercel. **Вернись сюда после части 2.** |

⚠️ **Не задавай `PORT`** — Railway подставляет порт сам.

### Шаг 1.4 — задеплоить

1. После сохранения переменных Railway сам запустит билд (или жми **Deployments → Deploy**).
2. Логи деплоя видны в реальном времени. Что должно появиться в конце:
   ```
   potok-backend слушает http://localhost:NNNN
   Worker pool : concurrency=2
   [recovery] ...
   ```
3. Если падает — смотри логи. Самые частые ошибки:
   - **«Не задана переменная окружения X»** — пропустил переменную, добавь и жми **Redeploy**.
   - Любая ошибка от ffmpeg или openai на старте — обычно опечатка в ключе. Проверь.

### Шаг 1.5 — получить публичный URL

1. Settings сервиса → раздел **Networking** → **«Generate Domain»**.
2. Railway выдаст что-то вроде `potok-backend-production.up.railway.app`.
3. **Скопируй и сохрани** — этот URL пойдёт в переменные Vercel в части 2.
4. Проверь, что бэкенд жив: открой `https://потвой-url/health` в браузере. Должно ответить:
   ```json
   {"ok":true,"service":"potok-backend"}
   ```

## Часть 2. Фронтенд на Vercel

### Шаг 2.1 — создать проект из GitHub

1. [vercel.com/new](https://vercel.com/new) → **Import Git Repository**.
2. Выбери репо `tokarevexplorer-svg/potok`. Если не видно — **«Adjust GitHub App Permissions»** и дай доступ.
3. На экране **«Configure Project»**:
   - **Root Directory** → нажми **Edit** → выбери `frontend/`.
   - **Framework Preset** должен сам определиться как **Next.js**. Если нет — выбери вручную.
   - **Build Command**, **Output Directory**, **Install Command** — оставь по умолчанию.

### Шаг 2.2 — переменные окружения

На том же экране (или потом в Settings → Environment Variables) добавь три:

| Имя | Значение |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | URL Supabase (тот же, что в бэкенде) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | **anon**, не service role. Из Supabase → API |
| `BACKEND_URL` | URL Railway-бэкенда из шага 1.5, **с `https://` и БЕЗ слэша в конце** |

У каждой переменной должно стоять **«Production and Preview»** — это значение по умолчанию. (Старого отдельного «Development» в Vercel UI больше нет — он был только для команды `vercel dev`, которой мы не пользуемся.)

### Шаг 2.3 — задеплоить

1. Жми **Deploy**.
2. Билд занимает 1–2 минуты. По завершении Vercel покажет URL вида `https://potok-xxx.vercel.app` (или твой кастомный субдомен).
3. **Скопируй URL** — он нужен для CORS на бэкенде.

### Шаг 2.4 — открыть и убедиться, что страница грузится

1. Открой URL фронта в браузере.
2. Главная редиректит на `/blog/references`.
3. **Если в таблице видны видео из локальной разработки — Supabase подключился.** Это нормально, БД одна на всё.
4. **Если ошибка** — открой DevTools → Console. Скорее всего, опечатка в env-переменной. Поправь в Vercel → Settings → Environment Variables → **Redeploy**.

## Часть 3. Замкнуть CORS (важно!)

Сейчас если ты на проде нажмёшь «Добавить видео», оно сохранится в БД, но обработка не запустится — бэкенд отбросит запрос с ошибкой CORS. Нужно разрешить домен Vercel.

1. Открой Railway → сервис бэкенда → **Variables**.
2. Найди `FRONTEND_ORIGINS` (или создай, если пропустил в шаге 1.3).
3. Значение — два URL через запятую, без пробелов:
   ```
   https://твой-vercel-url.vercel.app,http://localhost:3000
   ```
   (`localhost:3000` оставляем, чтобы локальный фронт продолжал работать с прод-бэкендом, если понадобится).
4. Сохрани → Railway автоматически перезапустит сервис.

## Часть 4. Проверка end-to-end

1. Открой Vercel-URL **на телефоне** (или с другого устройства).
2. Логин не нужен — приложение пока без auth.
3. Жми **«Добавить видео»** → вставь ссылку на любой Reels → **Анализировать**.
4. Через 30–60 сек обнови страницу — должна появиться обложка, автор, статистика.
5. Через ещё 30–60 сек — транскрипция и AI-саммари.

Если на каком-то шаге пусто:
- Открой Railway → сервис → **Logs**. Там увидишь весь процесс обработки или ошибку.
- Если строка зависла в «обрабатывается…» — наведи на неё, в `title` будет текст ошибки (или жди — Apify иногда отвечает дольше минуты).

## Дальше

- **Кастомный домен**: настраивается в Vercel → Settings → Domains. Купить домен можно у любого регистратора, в Vercel ввести и следовать инструкциям. Не обязательно сейчас.
- **Каждый push в `main` теперь авто-деплоится** и на Vercel, и на Railway. Это поведение по умолчанию.
- **Обновить URL в CLAUDE.md**: после первого успешного деплоя запиши Vercel-URL и Railway-URL в раздел «Текущая задача → Сессия 13» в CLAUDE.md, чтобы не потерять.

Если что-то пошло не так — открой Claude Code, вставь логи Railway/Vercel, разберёмся.
